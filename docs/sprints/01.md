# Sprint 01 — Structured ReAct Step Schema (Phase 1, Fake LLM)

## Goal

Introduce the internal JSON schema the agent uses to structure each reasoning step.
The fake LLM returns hardcoded JSON responses in the Phase 1 schema format so the parsing
and display logic can be built and tested without Ollama.

## As a user

As an employee of the company, I want the agent to show me not just the answer,
but also how it arrived at it — what it was thinking and what action it took —
so I can understand and trust the reasoning.

## Done when

- The fake LLM returns a JSON object with the Phase 1 fields: `thought`, `action`, `input`, `answer`
- The agent parses the JSON and extracts the `answer` field to display as plain text to the user
- The `thought`, `action`, and `input` fields are printed as an internal trace (e.g. prefixed with `[trace]`) so they are visible during development but clearly separated from the user-facing answer
- Malformed JSON from the fake LLM produces a clear error message rather than a crash
- The REPL behavior from Sprint 00 is unchanged from the user's perspective
