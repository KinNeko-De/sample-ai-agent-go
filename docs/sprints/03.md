# Sprint 02 — Streaming Response Display

## Goal

Instead of waiting for the full response before printing, display tokens as they arrive from Ollama.
This makes the agent feel responsive, especially for longer answers.

## As a user

As an employee of the company, I want to see the answer appear word by word as it is generated, so I don't have to stare at a blank screen while waiting for a long response.

## Done when

- Typing a message shows the response appearing token by token in the terminal
- The fake LLM streaming also works so the sprint is testable without Ollama
