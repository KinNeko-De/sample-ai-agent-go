# Sprint 03 — Tool Calling with Mock Data

## Goal

Introduce tool calling: the agent can decide to call a tool instead of answering directly,
and then use the tool result to form its final answer.
All tools return hardcoded mock data — no real backend needed.

## As a user

As an employee of the company, I want to ask questions that require live data — for example "What is the current management hierarchy?" — and get an up-to-date answer. The agent should recognize that this cannot be answered from general knowledge and must look it up using a tool.

## Done when

- Asking a question that requires the mock tool causes the agent to call the tool and incorporate the result into its answer
- Asking a question that does not require a tool gets a direct answer without any tool call
- After each tool execution, the tool result is appended to the message history as `{role: "tool", content: "..."}` before the next LLM call
